STEP 6 / 9

深度學習分割

傳統方法處理不了相黏細胞、不規則形狀、複雜組織。DL 分割器現在能跨顯微術式泛化，幾乎不用重訓——已是細胞 / 核 / 組織區段的預設首選。

Classical methods fail on touching cells, irregular morphologies, and crowded tissue. DL segmenters now generalize across modalities with little or no retraining — the default for cells, nuclei, and tissue compartments.

觀念

一、語意 / 實例 / 全景

🎨 語意

每個像素分到一個類別（細胞 / 背景 / 細胞核 / 細胞質）。不分個體。例：U-Net 輸出。

Per-pixel class label (cell / background / nucleus / cytoplasm). No individual identity. E.g., U-Net output.

🔢 實例

每個物件一個唯一 ID（cell 1, cell 2 …）。是下游所有量化的前提。Cellpose, StarDist, Mask R-CNN 都輸出實例分割。

Each object has a unique ID (cell 1, cell 2 …). Prerequisite for downstream quantification. Cellpose, StarDist, Mask R-CNN output instances.

🎭 全景

語意 + 實例混合。「可數物件」(thing) 用實例分；背景區域 (stuff) 用語意分。多細胞組織分析常見。

Semantic + instance combined. "Things" get instance IDs; "stuff" (background regions) gets semantic labels. Common in multi-cell tissue analysis.

決策樹

三、選哪個模型？

🌳 DL 分割選擇樹

Q1:
是密集圓核（如組織切片 DAPI）？→ StarDist 2D_versatile_fluo 或 StarDist H&E
Q2:
是一般細胞（活細胞、培養皿）？→ Cellpose-SAM (cyto3 模型)；先零訓練試試
Q3:
是多重染色組織（CODEX/IMC）？→ Mesmer（記得 channel 0 = nucleus, channel 1 = membrane）
Q4:
是細長 / 分支（細菌、神經元）？→ Omnipose
Q5:
是 H&E 病理 / 醫學影像（CT/MRI），且需自訂任務？→ nnU-Net（自配置，要花時間訓練）
Q6:
所有 pretrained 都不行？→ 收集 50–100 張標註，Cellpose human-in-the-loop fine-tune（GUI 友好）或 nnU-Net
Q1:
Densely packed round nuclei (DAPI in tissue)? → StarDist 2D_versatile_fluo or StarDist H&E
Q2:
General cells (live, culture)? → Cellpose-SAM (cyto3 model); try zero-shot first
Q3:
Multiplexed tissue (CODEX/IMC)? → Mesmer (channel 0 = nucleus, channel 1 = membrane)
Q4:
Elongated / branching (bacteria, neurons)? → Omnipose
Q5:
H&E / medical (CT/MRI), custom task? → nnU-Net (self-configuring, training required)
Q6:
None of the pretrained work? → Annotate 50–100 images and fine-tune Cellpose via human-in-the-loop or nnU-Net
💡
原則：先試 pretrained 通用模型零訓練（Cellpose-SAM cyto3、StarDist versatile）。大多數人從來不需要自己訓練。
真要訓練時，先用少量標註 + human-in-the-loop，再考慮全量訓練。 Principle: Try a pretrained generalist (Cellpose-SAM cyto3, StarDist versatile) zero-shot first. Most users never train.
When you must train, start with a few labels + human-in-the-loop before full training.
評估

四、分割結果評估

  • IoU (Intersection over Union, Jaccard)：交集 / 聯集，0–1 越高越好
  • Dice (F1)：2·交集 / (預測+真值)，與 IoU 同調但更敏感
  • AP@IoU：物件層級平均精度，IoU 閾值不同會得不同分數 (常見 0.5, 0.75)
  • Panoptic Quality (PQ)：偵測 + 分割綜合分數
  • Cell Tracking Challenge SEG：每個 GT 物件與最大重疊預測的 IoU 平均

務必在從未見過的測試集上評估。在訓練集上看「眼睛感覺準」是學習曲線一定會遇到的陷阱。

  • IoU (Jaccard): intersection over union, 0–1, higher is better
  • Dice (F1): 2·intersection / (pred + GT), more sensitive than IoU
  • AP@IoU: object-level average precision at given IoU thresholds (commonly 0.5, 0.75)
  • Panoptic Quality (PQ): combined detection + segmentation score
  • Cell Tracking Challenge SEG: mean IoU between each GT and best-overlap prediction

Always evaluate on a held-out test set. "It looks right by eye on the training set" is a trap every learner falls into once.

五、實作範例

# Cellpose 4 (Cellpose-SAM) — 2025 generalist
from cellpose import models, io
import numpy as np

img = io.imread("cells.tif")                     # [H, W] or [H, W, C]

# Zero-shot inference
mdl = models.Cellpose(gpu=True, model_type="cyto3")
masks, flows, styles, diams = mdl.eval(
    img,
    diameter=None,                             # auto-estimate
    channels=[0, 0],                           # [cyto, nuc] channel indices
    flow_threshold=0.4,
    cellprob_threshold=0.0
)
print(masks.max(), "cells found")
