去卷積 (Deconvolution)

顯微鏡會用「點擴散函數 (PSF)」模糊真實樣本。去卷積在數學上反轉這個模糊，讓影像更銳利、更可量化——3D widefield 與共軛焦尤其有感。

A microscope blurs reality by its point-spread function (PSF). Deconvolution inverts that blur mathematically — sharper, more quantitative images, especially in 3D widefield and confocal.

觀念

一、PSF 與影像形成模型

點擴散函數 (Point Spread Function, PSF)：一個亞解析度的光點（如小於繞射極限的螢光珠）在顯微鏡下不會仍是「一個點」，而會擴散成一個有特定形狀的模糊斑——這個斑就是 PSF。

影像形成的線性模型：

image = sample ⊗ PSF + noise

(⊗ 表示卷積)。去卷積就是已知 PSF（測量或理論），反推 sample。

The Point Spread Function (PSF) is the image of a sub-resolution point (e.g., a sub-diffraction-limited fluorescent bead) — it appears not as a point but as a blurred spot with a characteristic shape. That spot is the PSF.

Linear image-formation model:

image = sample ⊗ PSF + noise

(⊗ denotes convolution). Deconvolution recovers sample from a known PSF (measured or theoretical).

💡
關鍵直覺：PSF 在 XY 方向通常是 Gaussian-like，但在 Z 方向是不對稱的「沙漏 / 雙錐」形狀。這也是為什麼 3D 去卷積比 2D 重要得多——3D widefield 的離焦光是訊號的主要敵人。 Key intuition: The PSF is roughly Gaussian in XY but asymmetric ("hourglass / double cone") in Z. That is why 3D decon matters far more than 2D — in 3D widefield, out-of-focus light is the dominant enemy.
演算法

二、去卷積演算法

演算法 原理 優缺點
Inverse filter直接做 FFT 除法最快；對雜訊敏感，幾乎不堪用Direct FFT divisionFastest; extremely noise-sensitive, rarely usable
WienerFFT 域加入雜訊功率譜正則化快、穩定；無迭代FFT-domain regularization by noise power spectrumFast and stable; non-iterative
Richardson-Lucy (RL)最大概似迭代；Poisson 雜訊模型事實上的標準；超過 30 iter 噪聲放大Maximum-likelihood iteration with Poisson noise modelDe facto standard; noise amplifies past ~30 iter
RL-TVRL + Total Variation 正則化減少 ringing 與雜訊放大；多一參數RL with Total Variation regularizationReduces ringing & noise amplification; extra λ
Blind deconvolution同時估 PSF 與 sample無需測 PSF；不穩定，需良好初值Estimate PSF & sample jointlyNo PSF measurement needed; unstable, sensitive to init
CARE (DL)監督式 CNN 學 noisy/blurry→clean 對映最佳結果；需訓練配對資料Supervised CNN learning noisy/blurry → clean mappingBest results; requires paired training data

決策

三、要不要去卷積？

🌳 去卷積決策樹

Q1:
顯微術式？→ Widefield 3D：強烈建議。共軛焦：建議。Light-sheet：強烈建議 (multi-view 融合)。STED / SIM：通常不需。
Q2:
有 sub-resolution bead 影像可量 PSF 嗎？→ 有 → 用測量 PSF（最準）。沒有 → 用 PSFGenerator 算理論 PSF（Born-Wolf / Gibson-Lanni）
Q3:
要做強度量化嗎？→ 要 → 慎用！RL 不保線性，建議改用 Wiener（線性）或加 TV 正則
Q4:
有大量訓練配對資料嗎？→ 有 → CARE / DL decon，效果最好；沒有 → RL-TV 是黃金標準
Q1:
Modality? → Widefield 3D: strongly recommended. Confocal: recommended. Light-sheet: strongly recommended for multi-view fusion. STED/SIM: usually not needed.
Q2:
Bead images for PSF measurement available? → Yes → use measured PSF. No → compute theoretical PSF (Born-Wolf / Gibson-Lanni) via PSFGenerator.
Q3:
Will you quantify intensity? → Yes → caution! RL is not linearity-preserving; consider Wiener (linear) or RL with TV regularization.
Q4:
Lots of paired training data? → Yes → CARE / DL decon, best results. No → RL-TV is the gold standard.
⚠️
順序很重要：在 widefield，必須先做背景減除（移除離焦光的低頻成分）再做去卷積。否則離焦光會被「銳化」成偽訊號。 Order matters: In widefield, subtract background first (remove out-of-focus low-frequency content), then deconvolve. Otherwise the haze gets "sharpened" into pseudo-signal.
程式碼

四、去卷積實作

from skimage import restoration, io
import numpy as np

img = io.imread("stack3d.tif").astype("float32")
psf = io.imread("psf_measured.tif").astype("float32")
psf /= psf.sum()                                  # Normalize PSF

# Richardson-Lucy 3D (CPU, slow for big stacks)
deconv = restoration.richardson_lucy(img, psf, num_iter=20)

# Wiener (linear, single shot)
deconv_w = restoration.wiener(img, psf, balance=0.1)

# GPU-accelerated RL via flowdec (TensorFlow)
from flowdec import data as fd_data, restoration as fd_rest
algo = fd_rest.RichardsonLucyDeconvolver(n_dims=3).initialize()
res = algo.run(fd_data.Acquisition(data=img, kernel=psf), niter=25)
out = res.data
// Fiji + DeconvolutionLab2 plugin
// 1. Install: Help > Update > Manage update sites > DeconvolutionLab2
// 2. Open image and PSF

run("DeconvolutionLab2 Run",
    "-image platform 1 -psf platform 2 -algorithm RL 25 -out stack OUTPUT.tif");

// Or use PSFGenerator first to make a theoretical PSF
run("PSF Generator");
// → pick Born & Wolf 3D, set NA, lambda, n_immersion, pixel size, Z step
# CARE — content-aware restoration (DL)
from csbdeep.models import CARE
from csbdeep.utils import normalize

# Load pretrained model (e.g., from BioImage Model Zoo)
model = CARE(config=None, name="care_planaria_v1", basedir="models")

# Predict

from csbdeep.models import Config, CARE

X,Y,_ = create_patches(raw, patch_size=(16,64,64), n_patches_per_image=128)

model = CARE(config, "my_care"); model.train(X, Y)
