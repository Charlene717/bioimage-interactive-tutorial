配準與拼接

tile scan、多視角光片、多通道、長時間 live-cell——這些都需要把不同影像精準對齊到同一座標系。配準錯了，所有下游量化都會跟著錯。

Tile scans, multi-view light-sheet, multi-channel imaging, long live-cell time-lapses — all need precise alignment to a common coordinate system. Bad registration propagates into every measurement.

開場故事 · STORY

從 MRI 到 light-sheet：對齊影像 40 年

1992 年 Paul Viola 與 William Wells 在 ICCV 發表 Mutual Information，把資訊理論帶進醫學影像對齊——從此跨模態 (MRI vs CT) 配準有了金標準。

1996 年 Stephan Preibisch 在 Dresden 寫的 Fiji stitching 把同樣的數學帶到顯微 tile scan；2019 BigStitcher 把它延伸到 TB 級 light-sheet。從天文照片 (1980 IRAF) 到大腦圖譜 (Allen Brain Atlas 2017)，align 一直是「讓不同來源的影像說同一種語言」。

In 1992 Paul Viola and William Wells introduced Mutual Information at ICCV, bringing information theory into medical image registration — the gold standard for cross-modal (MRI ↔ CT) alignment.

Stephan Preibisch's Fiji stitching (1996) brought the same maths to microscopy tile scans; BigStitcher (2019) scaled it to TB-level light-sheet. From astronomy (IRAF 1980) to the Allen Brain Atlas (2017), registration is always about 'making different images speak the same language.'

變換家族

一、變換家族

🔲 Translation

只有 2 個參數 (dx, dy)，最簡單。常用於 time-lapse drift 校正、相機抖動、stage 飄移。

2 parameters (dx, dy). Used for time-lapse drift, camera jitter, stage drift.

🔄 Rigid

平移 + 旋轉，3–6 參數。適合同一樣本不同視角、染色對齊。

Translation + rotation, 3–6 parameters. Good for cross-view registration of the same sample.

📐 Affine

加入縮放與剪切，6（2D）或 12（3D）參數。處理輕微比例差異，例如不同顯微鏡。

Adds scaling and shearing. 6 params (2D) or 12 (3D). Handles small scale differences across microscopes.

🌊 B-spline 形變

非剛性變形，數百到數千參數。處理組織形變、CLEM、跨樣本對齊。

Non-rigid deformation, hundreds to thousands of params. For tissue deformation, CLEM, cross-sample alignment.

♾️ SyN 形變

ANTs 招牌算法，保證形變可逆。腦圖譜對齊的事實標準。

ANTs' flagship algorithm, guarantees invertibility. De facto standard for brain atlas registration.

📍 Landmark-based

手動指定對應點，TPS / BigWarp 重新內插。適合圖片差異大、自動方法失敗的情境（如 CLEM）。

Manually picked correspondences, TPS / BigWarp interpolation. For cases where auto methods fail (e.g., CLEM).

相似度指標

二、配準的核心元件

變換模型

從上面的家族中挑選。原則：能用簡單的就別用複雜的。過度參數化會 overfit 雜訊。

Pick from the family above. Principle: simplest that works. Over-parameterizing overfits noise.

相似度指標

SSD / NCC：同一模態。Mutual Information (MI)：跨模態（fluo + brightfield、CT + MRI）。NMI：MI 正規化版，更穩定。

SSD / NCC: same modality. Mutual Information (MI): cross-modal (fluo + brightfield, CT + MRI). NMI: normalized variant, more stable.

優化器

梯度下降、L-BFGS、(1+1) 演化、Powell。多解析度金字塔（先粗後細）幾乎永遠該開。

Gradient descent, L-BFGS, (1+1) evolutionary, Powell. Multi-resolution pyramid (coarse → fine) should almost always be enabled.

內插法

線性 (linear)：快、模糊。Cubic B-spline：較準但慢。Nearest-neighbour：保留標籤值（label image 必用）。

Linear: fast, slightly blurry. Cubic B-spline: more accurate, slower. Nearest-neighbour: must-use for label images.

拼接

三、Tile Stitching

顯微鏡視野有限，要看一整個組織切片，要做 tile scan（例如 10×10 個 tile，每個有 10% overlap）。Stitching 用 overlap 區域算出每個 tile 的精確位置。

Phase correlation：在 FFT 域算兩 tile 的相對位移，亞像素精度。Fiji Grid/Collection Stitching、MIST、BigStitcher 都用此原理但各有強項：

A microscope has a limited field of view; to see a whole tissue section you do a tile scan (e.g., 10×10 tiles with ~10% overlap). Stitching computes each tile's precise position from the overlap regions.

Phase correlation: FFT-domain estimation of tile-to-tile shifts with subpixel accuracy. Fiji Grid/Collection Stitching, MIST, and BigStitcher all rest on this principle:

⚠️
陷阱：stitching 工具仰賴顯微鏡 metadata 中的 stage 座標當初值。如果 stage 寫的座標是垃圾（不少老顯微鏡如此），請設成「regular grid」模式由演算法純從影像 refine。 Pitfall: Stitchers seed alignment from stage metadata. If your microscope writes garbage coordinates, switch to "regular grid" mode and let the algorithm refine from image content alone.

Q1:
Time-lapse drift? → Translation (phase_cross_correlation or pystackreg)
Q2:
Multi-channel (different lasers on the same sample)? → Rigid + MI (intensity distributions differ)
Q3:
Tile scan? → Per-tile translation handled by the stitcher
Q4:
Cross-sample / atlas? → Affine init → B-spline / SyN
Q5:

# 1) Time-lapse drift: phase correlation
from skimage.registration import phase_cross_correlation
from scipy.ndimage import shift
shifts, _, _ = phase_cross_correlation(frame_ref, frame_t, upsample_factor=10)
aligned = shift(frame_t, shift=shifts)

# 2) Multi-channel rigid with mutual information (SimpleITK)
import SimpleITK as sitk
fix = sitk.GetImageFromArray(ch_dapi); mov = sitk.GetImageFromArray(ch_gfp)
reg = sitk.ImageRegistrationMethod()
reg.SetMetricAsMattesMutualInformation(numberOfHistogramBins=50)
reg.SetOptimizerAsGradientDescent(learningRate=1.0, numberOfIterations=200)
reg.SetInitialTransform(sitk.Euler2DTransform())
reg.SetInterpolator(sitk.sitkLinear)
tx = reg.Execute(fix, mov)
moved = sitk.Resample(mov, fix, tx, sitk.sitkLinear)

# 3) Atlas-style deformable: ANTsPyX SyN
import ants
fix = ants.from_numpy(brain_atlas); mov = ants.from_numpy(my_brain)
out = ants.registration(fixed=fix, moving=mov, type_of_transform="SyN")
warped = out["warpedmovout"]

# 4) Stack-time stabilization with pystackreg
from pystackreg import StackReg
sr = StackReg(StackReg.RIGID_BODY)
stack_aligned = sr.register_transform_stack(stack, reference="previous")
// BigStitcher workflow (Fiji)
// 1. Plugins > BigStitcher > Define dataset
//    - Choose "multiple 2D/3D image stacks"
//    - Point to your CZI / OME-TIFF tiles

// 2. Plugins > BigStitcher > Stitching wizard
//    - Pairwise shift calculation: phase correlation
//    - Filter pairwise shifts: r > 0.7
//    - Optimize globally

// 3. Plugins > BigStitcher > Image fusion
//    - Output: 3D TIFF or HDF5/N5
//    - Blending: linear blending across overlaps

// For simple 2D grid stitching:
run("Grid/Collection stitching", "type=[Grid: row-by-row] order=[Right & Down] grid_size_x=10 grid_size_y=10 tile_overlap=10 first_file_index_i=0 directory=[/path] file_names=tile_{iii}.tif fusion_method=[Linear Blending] regression_threshold=0.30 max/avg_displacement_threshold=2.50 absolute_displacement_threshold=3.50 computation_parameters=[Save memory (but be slower)] image_output=[Fuse and display]");

公式拆解

Mutual Information 怎麼算

Σ 逐步推導
MI(A,B) = Σ p(a,b) · log[ p(a,b) / (p(a)·p(b)) ]
① 聯合直方圖Joint histogram 量化 A、B 強度到 32–64 bin，數每對 (a,b) 出現次數 → 聯合機率 p(a,b)。Quantize A and B to 32–64 bins, count (a,b) pair occurrences → joint probability p(a,b).
② 邊際機率Marginals p(a) = Σ_b p(a,b)；p(b) = Σ_a p(a,b)。p(a) = Σ_b p(a,b); p(b) = Σ_a p(a,b).
③ 資訊重疊Information overlap 若 A、B 完全獨立：p(a,b) = p(a)p(b) → log = 0 → MI = 0。對齊好時 MI 最大。If A and B are independent: p(a,b) = p(a)p(b) → log = 0 → MI = 0. MI peaks at best alignment.
④ 跨模態優勢Cross-modal advantage DAPI 與 GFP 強度分佈不同但生物結構相同 → SSD 失效但 MI 仍能找到「結構同步」。DAPI vs GFP have different intensity distributions but matching biology → SSD fails, MI still finds 'structural sync.'
為了 robust 化，常用 NMI = (H(A)+H(B)) / H(A,B)。NMI 對 overlap 大小較不敏感，是 BigStitcher 與 SimpleElastix 預設。
For robustness, use NMI = (H(A)+H(B)) / H(A,B). NMI is less sensitive to overlap size — the default in BigStitcher and SimpleElastix.
思辨討論

配準三大爭論

A. SSD 何時失敗、MI 何時必要？
你要對齊「同細胞」的 DAPI 與 GFP。SSD 與 MI 哪個比較合理？
Aligning DAPI vs GFP of the same cell — SSD or MI?
提示：DAPI 強度集中在核、GFP 強度分佈廣，強度差異大。SSD 會被強度主導，MI 抓「結構共同」。
Hint: DAPI is nucleus-centered, GFP is broad. SSD gets dominated by intensities; MI captures 'structural co-occurrence.'
B. Stitcher 的 stage metadata 信不過？
你的舊顯微鏡寫的 stage 座標誤差 50 μm。BigStitcher 怎麼用？
Your old microscope writes stage coords with 50 μm error. How do you run BigStitcher?
提示：用 'regular grid' 模式，完全不信 stage，純從影像 phase correlation 計算位置。
Hint: use 'regular grid' mode — distrust stage entirely and compute positions purely from phase correlation.
C. CLEM 為何自動方法常失敗？
光學影像與電子顯微影像對同一細胞拍——自動 MI 配準幾乎不工作。為什麼？
Light + electron microscopy of the same cell — automatic MI almost never works. Why?
提示：兩種模態看到的「結構」完全不同：光只看螢光標誌，EM 看膜邊界。需要 landmark + TPS 手動建立對應。
Hint: the two modalities reveal totally different 'structures' — fluorescence labels vs membrane boundaries. Use landmark + TPS for manual correspondences.
精選案例

兩個傳奇配準故事

① ASHLAR：CyCIF 多輪染色救星
Muhlich et al., Bioinformatics 2022 · DOI: 10.1093/bioinformatics/btac544

CyCIF (Cyclic Immunofluorescence) 在同一張切片做 20+ 輪 staining-imaging-bleaching 循環，但每次取下/放回 sample stage 都會位移幾微米。Sorger Lab 發表 ASHLAR：自動跨輪對齊 + 拼接，無需手動標 landmark。從此 multiplex IHC 在臨床病理可行——一張切片同時看 40+ 蛋白標誌。

CyCIF (Cyclic Immunofluorescence) runs 20+ stain-image-bleach cycles on one slide, but every stage cycle shifts a few microns. The Sorger Lab released ASHLAR: automated cross-round alignment + stitching without manual landmarks. Multiplex IHC in clinical pathology became feasible — 40+ protein markers on one slide.

啟示：把「手動但精準」變成「自動可重現」，往往就是讓一個 assay 進入主流的關鍵。
Lesson: turning 'manual but accurate' into 'automatic and reproducible' is often what brings an assay mainstream.
② Allen Brain Atlas：每隻小鼠腦對到「共同座標」
Allen Institute · Common Coordinate Framework (CCF) v3 2017

Allen Institute 為了讓全球研究者能直接比較「我的腦切片 vs 別人的」，建立 Common Coordinate Framework：每隻小鼠腦切片用 ANTs SyN 非線性配準到一個 reference brain。從此「左眶額皮質第 V 層」是全球統一座標。連帶 spatial transcriptomics、optogenetics 都受惠。

The Allen Institute built the Common Coordinate Framework so labs worldwide can compare 'my brain section vs theirs.' Every mouse brain section is nonlinearly registered to a reference brain via ANTs SyN. 'Left orbitofrontal cortex layer V' is now a globally consistent coordinate — benefiting spatial transcriptomics, optogenetics, and more.

啟示：基礎建設層的標準化往往比演算法本身重要——CCF 是神經科學的「通用語」。
Lesson: infrastructure-level standardization often matters more than the algorithm itself — CCF is neuroscience's lingua franca.

📜 歷史小教室： Viola & Wells 1992 把 MI 從資訊理論搬到醫學影像，靈感來自 Shannon 1948。論文當年只有 12 頁，卻奠定 30 年來醫學/生物影像 cross-modal registration 的根基。其中 Wells 還是 MGH 的影像引導手術 (image-guided surgery) 先驅——MI 真的被用來在手術房對齊術前 MRI 與術中超音波，救命級的應用。 Viola & Wells (1992) lifted MI from information theory (Shannon 1948) into medical imaging. A 12-page paper that's underpinned 30 years of cross-modal registration. Wells was also an image-guided-surgery pioneer at MGH — MI is literally used in operating rooms to align pre-op MRI with intra-op ultrasound. A life-saving application.
