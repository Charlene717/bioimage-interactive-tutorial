預處理與去噪

活細胞與低光顯微鏡為避免光毒性，必然光子受限——結果就是雜訊與不均勻照明。好的去噪 / 平場校正，往往決定下游分割是看到真細胞還是雜點。

Live-cell and low-light microscopy is photon-starved by necessity. Good denoising and shading correction often decide whether the segmenter sees real cells or speckle.

概覽

為什麼預處理重要？

顯微影像的雜訊主要來自三類：

  • Poisson (shot) noise：光子計數的統計波動，在低光時佔主導
  • Gaussian read noise：相機讀出電路的熱雜訊
  • Fixed-pattern / dark current：固定的相機缺陷

另一類更隱蔽的問題是不均勻照明（vignetting）：邊角的亮度比中心低，導致同一塊組織不同位置量出來的細胞「強度不同」。如果不校正，下游 marker 量化會變成「位置決定強度」的偽結果。

Microscopy noise has three main sources:

  • Poisson (shot) noise — photon-counting statistics, dominant in low light
  • Gaussian read noise — thermal noise in the camera readout
  • Fixed-pattern / dark current — persistent camera defects

A subtler problem is uneven illumination (vignetting): corners are dimmer than the centre, so the same tissue measured at different positions gives different intensities. Uncorrected, downstream marker quantification becomes a "position determines intensity" artefact.

去噪方法

一、經典去噪濾波器

🌫️

Gaussian

線性低通濾波，平均鄰域。適合：shot noise + 已稍亮的影像。缺點：模糊邊緣。
scipy.ndimage.gaussian_filter

Linear low-pass filter, averages a neighbourhood. Best for: shot noise on bright images. Downside: blurs edges.
scipy.ndimage.gaussian_filter

🧂

Median

非線性、保邊緣。適合：salt-and-pepper、hot pixels。缺點：對細小結構過度平滑。
skimage.filters.median

Non-linear, edge-preserving. Best for: salt-and-pepper, hot pixels. Downside: over-smooths thin structures.
skimage.filters.median

🪞

Bilateral / NLM

結合空間距離與灰階距離。保留邊緣且去高斯雜訊。缺點：慢。
skimage.restoration.denoise_nl_means

Combines spatial + intensity distance. Preserves edges while denoising. Downside: slow.
skimage.restoration.denoise_nl_means

🌀

BM3D

塊匹配 3D 變換。最強的傳統方法，PSNR 接近 DL。缺點：慢，參數多。
bm3d (PyPI)

Block-matching 3D transform. Strongest classical method — PSNR approaches DL. Downside: slow, many parameters.
bm3d (PyPI)

🤖

Noise2Void

自監督 DL：不需 clean ground truth，只需 noisy 影像本身。假設像素獨立雜訊。
n2v (CSBDeep)

Self-supervised DL — no clean ground truth needed, learns from noisy data alone. Assumes pixel-independent noise.
n2v (CSBDeep)

⚡

Cellpose 3

2025 推出的一鍵還原：denoise / deblur / upsample 三合一，內建多種 noise model。
cellpose.denoise

2025 release — one-click restoration combining denoise/deblur/upsample with multiple noise models.
cellpose.denoise

背景與平場校正

二、背景減除與平場校正

Rolling Ball（Sternberg 1983）：想像在影像「3D 表面」下滾一顆球，球心軌跡就是背景。半徑通常設為「最大前景物件直徑的 1.5 倍」。

BaSiC（Peng 2017）：從多張影像中估出平場 (flatfield) 與暗場 (darkfield)，特別適合 tile scan 與 HCS（每張影像看到的物體分佈不同，BaSiC 能反推照明場）。

Rolling Ball (Sternberg 1983): imagine rolling a ball under the "3D surface" of the image — the ball-centre trajectory is the background. Set the radius to ~1.5× the largest foreground object.

BaSiC (Peng 2017): estimates flatfield + darkfield from a collection of images, ideal for tile scans and HCS where every image sees a different object distribution.

⚠️
Rolling ball 半徑陷阱：太小會侵蝕真實訊號（細胞被當成背景的一部分減掉）；太大則保留不均背景。請先試 1.5× 最大物件直徑，再根據結果調整。 Rolling-ball radius pitfall: Too small erodes real signal (cells treated as background and subtracted); too large leaves uneven background. Start at 1.5× the largest object diameter, then adjust.
決策引導

三、預處理流程決策

🌳 預處理選擇決策樹

Q1:
是否要做量化（量強度、共定位）？→ 是 → 不要任意濾波！只做能被審稿人接受的步驟（背景減除 + 平場校正）
Q2:
背景明顯不均（tile scan、HCS、組織切片）？→ 是 → BaSiC 平場校正
Q3:
背景看似均勻但有緩慢梯度？→ 是 → Rolling Ball（半徑 = 1.5× 物件直徑）
Q4:
影像 SNR 太低，分割看不清細胞？→ 是 → 先試 Cellpose 3 denoise（一鍵），不行再 N2V fine-tune
Q5:
只是 hot pixels / 椒鹽？→ 是 → 3×3 median 就夠了，別動用 DL
Q1:
Will you quantify (intensity, colocalization)? → Yes → Don't filter casually. Use only reviewer-defensible steps (background subtraction + flat-field).
Q2:
Background obviously uneven (tile scan, HCS, tissue section)? → Yes → BaSiC flat-field correction
Q3:
Background mostly uniform but slow gradient? → Yes → Rolling Ball (radius = 1.5× object diameter)
Q4:
SNR too low to see cells for segmentation? → Yes → Try Cellpose 3 denoise first; if not enough, fine-tune N2V
Q5:
Just hot pixels / salt-and-pepper? → Yes → 3×3 median is enough; don't reach for DL
程式碼

四、預處理實作

import numpy as np
from skimage import io, filters, restoration, morphology

img = io.imread("cells.tif").astype("float32")

# 1) Median for hot pixels
img_m = filters.median(img, morphology.disk(2))

# 2) Rolling-ball background subtraction
bg = restoration.rolling_ball(img_m, radius=50)
img_bg = img_m - bg

# 3) Non-local means for Gaussian noise (edge-preserving)
sigma_est = np.mean(restoration.estimate_sigma(img_bg))
img_nlm = restoration.denoise_nl_means(img_bg, h=1.15*sigma_est, fast_mode=True)

# 4) BaSiC flat-field correction (multi-image)
from basicpy import BaSiC
basic = BaSiC(smoothness_flatfield=1.0)
basic.fit(stack)                              # stack: list of tile images
corrected = basic.transform(stack)

# 5) Noise2Void — self-supervised, no clean GT
from n2v.models import N2V, N2VConfig
config = N2VConfig(noisy_stack, unet_kern_size=3, train_epochs=50)
model = N2V(config, "my_n2v_model", basedir="models")
model.train(noisy_stack, noisy_stack)         # input == target intentionally
denoised = model.predict(noisy_stack, axes="YX")
// Fiji macro: rolling-ball background + median
run("Bio-Formats Importer", "open=[/path/cells.tif]");

// Median filter for hot pixels
run("Median...", "radius=2");

// Rolling-ball background subtraction
run("Subtract Background...", "rolling=50");

// BaSiC plugin (install from update site "BaSiC")
run("BaSiC ", "processing_stack=cells flat-field=None dark-field=None shading_estimation=[Estimate shading profiles] shading_model=[Estimate flat-field only (ignore dark-field)] setting_regularisationparametes=Automatic temporal_drift=Ignore correction_options=[Compute shading and correct images] lambda_flat=0.50 lambda_dark=0.50");
# Cellpose 3 — one-click denoise / deblur / upsample
from cellpose import denoise

# model_type: 'denoise_cyto3' | 'deblur_cyto3' | 'upsample_cyto3'
dn = denoise.DenoiseModel(model_type="denoise_cyto3", gpu=True)
restored = dn.eval(noisy_img, channels=[0,0])

# Or do it as part of segmentation in one call
from cellpose import models
mdl = models.CellposeDenoiseModel(gpu=True,
    model_type="cyto3", restore_type="denoise_cyto3")
masks, flows, styles, restored_img = mdl.eval(noisy_img, diameter=30)
陷阱

五、常見錯誤與陷阱

🚫
不要在量化分析前任意濾波。Gaussian 濾波會降低訊號峰值、改變共定位係數，論文審稿時很容易被質疑。要做的話，記錄所有參數並一致套用到對照組。 Never filter casually before quantification. Gaussian filtering lowers signal peaks, biases colocalization coefficients, and is a frequent reviewer target. If you must, document every parameter and apply identically to controls.
⚠️
Noise2Void 的隱含假設：像素間雜訊獨立。對於結構化雜訊（直線 artifact、固定花紋）會失效，請改用 Structured N2V 或 Noise2Self。 N2V hidden assumption: noise must be pixel-independent. Fails on structured noise (line artefacts, fixed patterns) — use Structured N2V or Noise2Self instead.
💡
最佳實踐順序：(1) 暗場校正（如有 dark frame）
(2) 平場校正 (BaSiC)
(3) Hot pixel 修正（小 median）
(4) 視 SNR 決定是否需 DL denoise
(5) 量化分析（記得用未濾波的影像或一致濾波的影像） Recommended order: (1) Dark-frame subtraction (if available)
(2) Flat-field correction (BaSiC)
(3) Hot-pixel fix (small median)
(4) DL denoise only if SNR demands it
(5) Quantify on unfiltered or consistently filtered images
