ADVANCED

工具比較與整合

選對工具 + 把它們可重現地串起來——這是一半的功夫。同一個生物問題可以用 Fiji macro、CellProfiler、napari + Python notebook、Snakemake on cluster 解決。沒有放諸四海皆準的「最佳」。

Choosing the right tool and wiring them together reproducibly is half the battle. The same biology can live in a Fiji macro, a CellProfiler pipeline, a napari + Python notebook, or a Snakemake-orchestrated workflow. None is universally best.

開場故事 · STORY

從 ImageJ 1997 到 BioImage.IO Chatbot 2024：工具生態 30 年

1997 年 Wayne Rasband 在 NIH 發布 ImageJ 1.0；2012 Pavel Tomančák 等發起 Fiji 「Is just ImageJ」分發版；2017 Nick Sofroniew 在 Janelia 為 Python 寫 napari N-D viewer；同時 Anne Carpenter 在 Broad 發展 CellProfiler。

20 年間 bioimage 工具從「個人英雄」走向「社群協作」。今天選 Fiji vs napari vs CellProfiler vs QuPath vs ilastik，常常不是「哪個比較強」，而是「哪個比較符合你的工作流」。2024 BioImage.IO Chatbot 進一步出現，AI 助你選工具找模型。

Wayne Rasband released ImageJ 1.0 at NIH in 1997; Pavel Tomančák and team launched Fiji 'Is just ImageJ' in 2012; Nick Sofroniew at Janelia wrote napari for Python in 2017; meanwhile Anne Carpenter at Broad built CellProfiler.

In 20 years bioimage tooling moved from 'individual heroes' to 'community collaboration.' Today the choice among Fiji / napari / CellProfiler / QuPath / ilastik is rarely 'which is stronger' but 'which fits your workflow.' By 2024 the BioImage.IO Chatbot arrived — AI helping you pick tools and find models.

何時離開 Fiji

二、何時離開 Fiji？

Fiji 完美的場景：單一影像互動探索、quick prototyping、教學示範、給生物學家用。
當以下任一項出現時，就該轉向 Python (napari + scikit-image + Cellpose + …)：

  • 需要版本控制（git）與 PR review
  • 需要叢集 / 雲端並行處理
  • 需要深度學習（雖 deepImageJ 存在，但訓練仍是 Python）
  • 需要與非影像資料（單細胞表達、臨床 metadata）整合分析
  • 需要可重現的 environment（conda lockfile / Docker）
  • 需要寫單元測試

Fiji's sweet spot: single-image interactive exploration, quick prototyping, teaching, handing off to biologists.
Switch to Python (napari + scikit-image + Cellpose + …) when any of these appear:

  • You need version control (git) and PR review
  • You need cluster / cloud parallelization
  • You need deep learning (deepImageJ exists but training still lives in Python)
  • You need to integrate with non-image data (single-cell expression, clinical metadata)
  • You need a reproducible environment (conda lockfile / Docker)
  • You need unit tests
可重現工作流

三、可重現工作流

「論文發表後三年，誰能重跑你的分析？」這是 reproducibility 黃金準則。

  • 環境：environment.yml（conda）/ pyproject.toml / uv.lock，或 Docker / Apptainer image
  • 工作流引擎：Snakemake（Python 路徑模式）、Nextflow（叢集 / 雲端友善）、CWL
  • 資料管理：DVC、bioimage 的 OME-Zarr + BIDS-Microscopy
  • 分享：BioImage Archive、IDR、BioImage Model Zoo、Zenodo (持久 DOI)
  • 標準：BAND (cloud bioimage desktop)、image.sc 論壇

"Three years after publication, can someone re-run your analysis?" — the gold standard of reproducibility.

  • Environment: environment.yml (conda), pyproject.toml, uv.lock, or Docker / Apptainer
  • Workflow engines: Snakemake (Python-path style), Nextflow (cluster/cloud-friendly), CWL
  • Data management: DVC, plus OME-Zarr + BIDS-Microscopy
  • Sharing: BioImage Archive, IDR, BioImage Model Zoo, Zenodo (persistent DOI)
  • Standards: BAND (cloud bioimage desktop), image.sc forum

公式拆解

Reproducibility 5 層模型

Σ 逐步推導
Reproducibility = env + container + workflow + data + DOI
① Layer 1：pinned environmentLayer 1: pinned environment conda.lock / pixi.lock / uv.lock 把版本鎖死。conda.lock / pixi.lock / uv.lock pin every version.
② Layer 2：containerLayer 2: container Docker / Apptainer image hash 確保 OS + lib 完全相同。Docker / Apptainer image hash guarantees identical OS + libs.
③ Layer 3：workflow engineLayer 3: workflow engine Snakemake / Nextflow DAG 自動記錄 step 依賴。Snakemake / Nextflow DAG auto-records step dependencies.
④ Layer 4：example dataLayer 4: example data 附 example dataset + expected output 給審稿驗證。Bundle an example dataset + expected output for reviewer verification.
⑤ Layer 5：persistent DOILayer 5: persistent DOI Zenodo DOI 永久保存；論文連結直接到此。Zenodo DOI permanently archives; papers link directly to it.
做到 Layer 1 = 最低限度；做到 Layer 5 = gold-standard reproducibility。多數 bioimage 論文目前在 Layer 2–3 之間。Layer 5 是 2025 之後 nature 系列期刊愈來愈高的標準。
Layer 1 is the minimum; Layer 5 is gold-standard reproducibility. Most bioimage papers sit between Layers 2 and 3 today. Layer 5 is increasingly required by Nature journals from 2025 onward.

精選案例

兩個工具生態的傳奇

① Wayne Rasband：30 年免費 ImageJ
NIH 1997–2027 · ImageJ / Fiji 引用 100,000+ 次

Wayne Rasband 1997 在 NIH 開始寫 ImageJ。從未收一分錢，從未發 startup。他 2024 退休前最後一次 commit 是 fix 一個 macro bug。30 年累積：ImageJ + Fiji 被引用 100,000+ 次，Schindelin 2012 Fiji Nat Methods 一篇就 30,000+ 引用。Rasband 是計算生物學最沉默卻最重要的英雄之一。

Wayne Rasband started writing ImageJ at NIH in 1997. Never charged a cent; never founded a startup. His last commit before retiring in 2024 was a macro bug fix. Thirty years: ImageJ + Fiji garnered 100,000+ citations; the Fiji 2012 Nat Methods paper alone has 30,000+. Rasband is computational biology's quietest yet most important hero.

啟示：open-source 的真實成本不是 money，是「持續維護 30 年」。這值得社會制度去保護。
Lesson: the real cost of open-source isn't money — it's 30 years of sustained maintenance. Society must protect that.
② BioImage.IO Chatbot：AI 助你選工具
Ouyang et al., Nat Methods 21:1325 (2024) · chat.bioimage.io

Wei Ouyang 等 2024 在 Nat Methods 發表 BioImage.IO Chatbot：community-driven AI 助手，問它「我有 confocal Z-stack 想做 cell tracking 該用什麼？」會給工具建議 + 模型推薦 + 範例 code。背後整合 BioImage Model Zoo + image.sc 知識庫。是 LLM 應用到 bioimage 教學的重要里程碑。

Wei Ouyang et al. published the BioImage.IO Chatbot in Nat Methods 2024: a community-driven AI assistant. Ask 'I have a confocal Z-stack and want to do cell tracking, what should I use?' and it suggests tools + recommends models + provides example code. It integrates the BioImage Model Zoo + the image.sc knowledge base — a landmark for LLM-in-bioimage education.

啟示：當工具生態複雜到沒人能全會，社群 + AI 助手成為新的「介面」。
Lesson: when tooling becomes too complex for any one person, community + AI assistants become the new interface.
