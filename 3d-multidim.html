3D 與多維分析

真實生物大多是 3D、多通道、有時間維度。3D 分割、各向異性處理、N-D 互動式檢視器、大資料流式運算——是 organoid、發育、light-sheet 工作流的基本門檻。

Most biology is 3D, multi-channel, and time-resolved. 3D segmentation, anisotropy handling, N-D viewers, and out-of-core computation are baseline for organoid, developmental, and light-sheet workflows.

各向異性

一、各向異性體素

典型共軛焦：XY = 0.21 µm、Z = 1.0 µm。這意味一個體素在 Z 方向比 XY 大 5 倍。

後果：

  • 3D 距離計算 / 球度 / 表面積必須乘上 voxel size
  • 3D 分割若不告訴模型 anisotropy，cell 在 Z 軸會被誤判得「過短或過長」
  • 視覺化必須開「scale = (z, y, x)」才能看到正確的細胞比例

兩種處理策略：

  • (A) 傳給演算法 anisotropy（Cellpose 的 anisotropy=5.0）
  • (B) resample 到 isotropic（用 zoom / scipy zoom），memory 換正確性

Typical confocal: XY = 0.21 µm, Z = 1.0 µm — a voxel is 5× larger in Z than XY.

Consequences:

  • 3D distance / sphericity / surface area must be scaled by voxel size
  • 3D segmentation will mis-judge cell length in Z without anisotropy info
  • Visualization must use "scale = (z, y, x)" to show correct proportions

Two strategies:

  • (A) Pass anisotropy to the algorithm (Cellpose's anisotropy=5.0)
  • (B) Resample to isotropic (zoom / scipy zoom) — trades memory for correctness
3D 分割

二、3D 分割策略

策略作法優缺
真正 3D3D CNN 一次處理整 stack最準（密集組織）；最耗記憶體3D CNN processes whole stackMost accurate (dense tissue); memory-heavy
2D + Stitch (IoU 跨 z)每片 2D 分割，跨 z 用 IoU 連同物件快；稀疏物件足夠；Cellpose stitch_threshold2D per slice, link by IoU across zFast; enough for sparse objects; Cellpose stitch_threshold
2.5D輸入 3 鄰片，輸出中片 mask折衷；nnU-Net 預設選項之一Input 3 neighbour slices, output middleCompromise; nnU-Net offers this

主流 3D 模型

  • Cellpose 3D：do_3D=True、anisotropy=5.0。一鍵啟動。
  • StarDist 3D：star-convex 多面體，圓核 3D 首選。
  • PlantSeg：植物 / 動物組織 3D 細胞邊界分割，CNN + graph partitioning。
  • nnU-Net 3D_fullres：自配置 3D U-Net；醫學影像 benchmark 標配。
  • Micro-SAM 3D：互動式 3D 分割，標一層即可向上下傳播。
大資料

三、處理大資料

Light-sheet 一次實驗常產生數百 GB 到 TB。一般 imread 整檔讀入 RAM 直接 OOM。解法是分塊存取與惰性運算。

  • OME-Zarr：多解析度金字塔的雲端原生格式（NGFF 標準），分塊 chunked。
  • N5 / HDF5：BigDataViewer 慣用，分塊隨機存取。
  • dask / dask-image：把陣列操作延後執行，scheduler 自動切塊與並行。
  • BigDataViewer / BigStitcher / mobie：TB 級互動視覺化。

A single light-sheet run often produces hundreds of GB to TB. Plain imread OOMs. Solution: chunked storage + lazy computation.

  • OME-Zarr: cloud-native pyramidal format (NGFF), chunked.
  • N5 / HDF5: BigDataViewer's preferred chunked storage.
  • dask / dask-image: defer execution; scheduler chunks + parallelizes automatically.
  • BigDataViewer / BigStitcher / mobie: TB-scale interactive visualization.
napari

四、napari

napari 是現代 Python bioimage 生態的互動中樞，原生支援 5D 影像、label layer、points / shapes / vectors / tracks / surfaces。豐富的 plugin 生態：

  • napari-cellpose、napari-stardist：分割
  • micro-sam：互動式提示分割
  • napari-bigstitcher：拼接
  • btrack / napari-arboretum：追蹤與譜系
  • napari-clemreg、napari-bigwarp：配準
  • napari-skimage-regionprops：互動量化

napari is the modern Python bioimage interactive hub — native 5D images, label/points/shapes/vectors/tracks/surfaces layers. Rich plugin ecosystem:

  • napari-cellpose, napari-stardist: segmentation
  • micro-sam: interactive promptable segmentation
  • napari-bigstitcher: stitching
  • btrack / napari-arboretum: tracking and lineage
  • napari-clemreg, napari-bigwarp: registration
  • napari-skimage-regionprops: interactive quantification

五、實作

# Cellpose 3D — passing anisotropy
from cellpose import models
mdl = models.Cellpose(gpu=True, model_type="cyto3")
masks, _, _, _ = mdl.eval(
    stack,                    # [Z, Y, X] or [Z, Y, X, C]
    diameter=30,              # in XY pixels
    do_3D=True,              # true 3D
    anisotropy=5.0,           # Z is 5x XY voxel size
    channels=[0, 0]
)

# Alternative: 2D + stitch (faster, sparse cells)
masks2, _, _, _ = mdl.eval(stack, diameter=30, do_3D=False,
                           stitch_threshold=0.4)
# napari viewer + dask for large stacks
import napari, dask.array as da
from dask_image.imread import imread

# Lazy loading — no memory pressure
stack = imread("slices/*.tif")        # dask array
labels = imread("masks/*.tif")

viewer = napari.Viewer()
viewer.add_image(stack, scale=(5, 1, 1),   # (z, y, x) anisotropy
                 contrast_limits=[100, 4000])
viewer.add_labels(labels, scale=(5, 1, 1))

# Interactively segment one slice with cellpose-napari plugin
napari.run()
# Convert big TIFF to OME-Zarr (chunked pyramidal)
import tifffile, zarr
from ome_zarr.writer import write_image
from ome_zarr.io import parse_url

img = tifffile.memmap("huge.tif")               # memmap = lazy
store = parse_url("out.ome.zarr", mode="w").store
root = zarr.group(store=store)
write_image(image=img, group=root, axes="tczyx",
            storage_options={"chunks": (1,1,64,512,512)},
            scaler=None)

# Read back lazily
g = zarr.open("out.ome.zarr", mode="r")
arr = g[0]                                  # highest-resolution level
chunk = arr[0, 0, 10:12, 100:200, 100:200]   # tiny read
