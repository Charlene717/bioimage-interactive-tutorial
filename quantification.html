量化與特徵

分割不是終點，量化才是。把每個物件變成一個「影像表型向量」——強度、形態、共定位、紋理——這就是 Cell Painting、表型分群、統計檢定的原料。

Segmentation isn't the goal — quantification is. Each object becomes an "image phenotype vector" (intensity, morphology, colocalization, texture) — the raw material for Cell Painting, phenotyping, and statistics.

共定位深探

二、共定位陷阱

共定位看似「兩通道一起亮多少」，但細節決定論文是否可信。

  • Pearson r 對線性偏移敏感：兩通道同步背景升高，r 不變；獨自背景升一個，r 大降——所以必須做背景減除。
  • Manders 需閾值：M1 = (有 ch2 信號的 ch1 強度) / (全 ch1 強度)。閾值選不對結果飄移嚴重 → 用 Costes 自動閾值。
  • Costes 隨機化檢定：把一個通道區塊化隨機洗牌，看 r 在隨機分佈中的位置。p < 0.05 才算顯著高於隨機共定位。
  • Object-based：對胞器級結構（粒線體、囊泡），先分割再算「重疊比例」比像素級更合理。

Colocalization looks like "how much do two channels co-light?" but the details decide whether the paper is trustworthy.

  • Pearson r is sensitive to linear offset: simultaneous background rise leaves r unchanged; rise in just one channel drops r dramatically → background-subtract first.
  • Manders requires a threshold: M1 = (ch1 intensity where ch2 is positive) / (total ch1). Bad threshold → unstable result → use Costes auto-threshold.
  • Costes randomization: shuffle blocks of one channel and see where r sits in the null distribution. Only p < 0.05 vs. random counts as significant.
  • Object-based: for organelle-level structures (mitochondria, vesicles), segment first then compute fractional overlap — more meaningful than pixel-level metrics.

# Per-cell Haralick texture (loop over labels)
def haralick_per_obj(img, labels, levels=32):
    img_q = (img.astype("float32")/img.max()*(levels-1)).astype("uint8")
    feats = []
    for lbl in np.unique(labels)[1:]:
        ys, xs = np.where(labels==lbl)
        if len(ys)<10: continue
        patch = img_q[ys.min():ys.max()+1, xs.min():xs.max()+1]
        glcm = graycomatrix(patch, [1], [0,np.pi/4,np.pi/2], levels=levels, symmetric=True, normed=True)
        feats.append({"label":lbl,
            "contrast":graycoprops(glcm,"contrast").mean(),
            "homog":graycoprops(glcm,"homogeneity").mean(),
            "correlation":graycoprops(glcm,"correlation").mean()})
    return pd.DataFrame(feats)
import numpy as np
from scipy.stats import pearsonr

# Background subtract first!
ch1 = ch1 - np.percentile(ch1, 5)
ch2 = ch2 - np.percentile(ch2, 5)
ch1 = np.clip(ch1, 0, None); ch2 = np.clip(ch2, 0, None)

# Pearson
r, p = pearsonr(ch1.ravel(), ch2.ravel())

# Manders with Costes auto-threshold
def costes_thresholds(c1, c2):
    flat1, flat2 = c1.ravel(), c2.ravel()
    # Orthogonal regression line through origin
    a = np.cov(flat1, flat2)[0,1] / np.var(flat1)
    b = flat2.mean() - a*flat1.mean()
    # Iterate down until Pearson on (T1,T2) area drops to ~0
    T1 = flat1.max()
    while T1 > 0:
        T2 = a*T1 + b
        mask = (flat1 < T1) | (flat2 < T2)
        if mask.sum() < 100: break
        rr, _ = pearsonr(flat1[mask], flat2[mask])
        if rr <= 0.01: break
        T1 -= 1
    return T1, T2

T1, T2 = costes_thresholds(ch1, ch2)
fg1 = ch1 > T1; fg2 = ch2 > T2
M1 = ch1[fg2].sum() / ch1.sum()    # ch1 in ch2+ region / total ch1
M2 = ch2[fg1].sum() / ch2.sum()

公式拆解

Costes 隨機化檢定 (Manders 共定位)

Σ 逐步推導
p = #{ robs ≤ rrandom,k } / N
① 觀測 rObserved r r_obs = Pearson(channel A, channel B)（建議先扣背景）。r_obs = Pearson(channel A, channel B) (after subtracting background).
② 區塊洗牌Block shuffle 把 channel B 分成 PSF 大小的小塊，隨機重排位置。Slice channel B into PSF-sized blocks and shuffle their positions.
③ null 分佈Null distribution 重複 N=200 次，得 r_random_k 分佈。Repeat N=200 times to build a null distribution of r_random_k.
④ p-valuep-value p = r_obs 落在 null 分佈右端的比例；p < 0.05 才算「顯著高於亂猜」。p = position of r_obs in the right tail; p < 0.05 = 'genuinely above random.'
⑤ Manders M1/M2Manders M1/M2 用 Costes 自動閾值 (T1, T2) 後算：M1 = Σ A(B≥T2) / Σ A；M2 對稱。After Costes auto-thresholds (T1, T2): M1 = Σ A(B≥T2) / Σ A; M2 symmetric.
結論：Pearson r 高 ≠ 共定位真實。永遠先扣背景 + Costes 檢定 + object-based 三步驗證。Manders M1/M2 需明確報告閾值方法。
Bottom line: a high Pearson r ≠ true colocalization. Always do background subtraction + Costes test + object-based as three layers of verification. Manders M1/M2 must explicitly report the threshold method.

量化三大爭論

A. Pearson r=0.9 ≠ 共定位？
你算出 r=0.9，是否就能說 protein A 與 protein B 共定位？

精選案例

兩個改寫量化的故事

① Manders vs Costes：共定位 10 年論戰
Manders 1993 J. Microsc. + Costes 2004 Biophys J

Manders 1993 提出 M1/M2 後迅速普及，但 90 年代末研究者發現：不同人選的閾值讓 M1 差異極大。Sylvain Costes 在 2004 Biophys J 提出「自動閾值 + randomization test」三重檢定。從此 colocalization 從「文藝復興」走向「實證學科」。Bolte & Cordelières 2006 J. Microsc. 整理整套 best practice，至今 30,000+ 引用。

Manders 1993 introduced M1/M2 — soon popular but by the late 1990s researchers found different thresholds produced wildly different M1. Sylvain Costes (Biophys J 2004) proposed an 'auto-threshold + randomization test' triple check. Colocalization moved from 'Renaissance' to 'empirical discipline.' Bolte & Cordelières (J. Microsc. 2006) summarized best practices — now 30,000+ citations.

啟示：方法論之爭往往讓一個領域成熟。「方便用 → 嚴謹用」是必經之路。
Lesson: methodological debate matures a field. 'Convenient' → 'rigorous' is a necessary path.
② JUMP-CP：3 百萬 Cell Painting 影像
Chandrasekaran et al., Nat Methods 21:1114 (2024) · jump-cellpainting.broadinstitute.org

Broad 領導的 12 institution JUMP-CP 聯盟，2024 在 Nat Methods 發表 3,072,156 張 Cell Painting 影像 × 12,602 化合物 + 8,069 基因擾動，全部用 CellProfiler 標準特徵 (1746 維)。後續 DeepProfiler v2 改用 UNI embedding 取代手工特徵，性能再提升。整個 image-based profiling 領域從此進入「大數據+ML」時代。

The Broad-led 12-institution JUMP-CP consortium published in Nat Methods 2024: 3,072,156 Cell Painting images × 12,602 compounds + 8,069 genetic perturbations, all processed with CellProfiler's standard 1,746-feature pipeline. DeepProfiler v2 later swapped UNI embeddings for hand-crafted features, lifting performance. Image-based profiling entered the 'big data + ML' era.

啟示：好的標準化資料集 > 任何單篇論文的演算法創新。
Lesson: a well-standardized dataset matters more than any single algorithmic paper.

Royal Society A 發表相關係數時，他研究的是「父母與孩子身高的關係」——完全沒考慮顯微影像背景。100 多年後同樣的 r 仍在每個 colocalization 報告中。歷史諷刺的是：當代統計學家 Pearson 本人會被自己的 r 騙到——因為他沒考慮「兩通道背景同步偏高」造成的偽 r。所以 Costes 才必須出現。 When Karl Pearson published his correlation coefficient in Phil. Trans. Royal Society A (1896), he was studying 'parents' and children's heights' — microscopy background was not on his mind. Over 100 years later, the same r still anchors every colocalization report. The historical irony: even Pearson, the master statistician, would have been fooled by his own r — because he hadn't anticipated 'both channels with joint high background' creating spurious r. That's why Costes had to come along.
